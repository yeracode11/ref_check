# Fridge Manager System

**Fridge Manager** — внутренняя система для дистрибьюторов напитков.  
Позволяет менеджерам отмечать посещения торговых точек, проверять холодильники, делать фото-отчёты и фиксировать геолокацию.  
Администраторы видят статистику, отчёты и контроль выполнения задач.

---

## Цели проекта

- Автоматизация контроля выездных менеджеров.
- Фиксация факта посещения холодильников с фото и координатами.
- Централизованный сбор данных по точкам.
- Упрощение отчетности и исключение ручного контроля.

---

## Основные функции

| Роль | Возможности |
|------|--------------|
|  **Менеджер** | Авторизация, список холодильников, отметка посещения, фото, геолокация |
|  **Администратор** | Управление пользователями и холодильниками, просмотр всех отметок, фильтры по датам и локациям, экспорт данных |

## Технический стек

**Backend:**
- [Node.js](https://nodejs.org) — среда выполнения JS  
- [Express.js](https://expressjs.com) — веб-фреймворк  
- [MongoDB](https://www.mongodb.com) — база данных (локально через Docker)  
- [Mongoose](https://mongoosejs.com) — ORM для MongoDB  
- [JWT](https://jwt.io) — авторизация (в планах)  
- [dotenv](https://www.npmjs.com/package/dotenv) — конфигурация окружения  
- [CORS](https://www.npmjs.com/package/cors) — разрешения между доменами

**Frontend (MVP план):**
- [Next.js](https://nextjs.org) / [React](https://react.dev)
- [Tailwind CSS](https://tailwindcss.com)
- [Axios](https://axios-http.com)
